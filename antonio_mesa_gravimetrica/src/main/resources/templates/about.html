Acerca del Proyecto

1. Presentación del Autor

Este proyecto ha sido diseñado, programado e implementado de forma íntegra por Antonio Garrido Bueno, en calidad de alumno de la Universidad de Jaén (UJA). La realización de esta plataforma surge como respuesta a la necesidad de digitalizar y optimizar los procesos de ensayo y análisis mineralúrgico dentro de los laboratorios de ingeniería, aplicando software industrial moderno con sistemas embebidos para el monitoreo en tiempo real.

A través de este desarrollo se unifican conocimientos de electrónica general, protocolos de transporte, IoT (Internet de las Cosas), ingeniería de servicios de telecomunicación, gestión de redes de telecomunicaciones, seguridad de redes de telecomunicaciones y redes basadas en dispositivos móviles.


2. Origen del Proyecto e Idea Inicial

Las mesas gravimétricas son equipos mecánicos críticos en la minería para la separación de minerales por densidades utilizando flujos de agua y vibraciones controladas. Sin embargo, la captura de datos tradicional en estos equipos suele ser analógica o manual, lo que introduce errores humanos, pérdida de precisión y dificultades para realizar un seguimiento histórico de los ensayos de laboratorio.

La meta inicial fue clara: instrumentar la mesa gravimétrica. Se propuso diseñar un ecosistema capaz de medir la inclinación estructural (Pitch y Roll), la frecuencia de sacudidas por segundo, la velocidad lineal del trayecto de vibración, el caudal de agua inyectado en la red de lavado y la masa acumulada en tiempo real.


3. Cronología del Desarrollo Paso a Paso

Fase I: Hardware y Programación del Sistema Embebido (ESP32)

El núcleo físico del proyecto se construyó sobre una red de microcontroladores ESP32 debido a su potencia de cómputo y su módulo WiFi nativo. En esta etapa:

  • Se conectó y calibró un acelerómetro digital ADXL345 a través del protocolo I2C para capturar la orientación espacial.
  • Se instalaron sensores de flujo electrónicos y una celda de carga para la masa.
  • Se implementaron finales de carrera para capturar las señales mecánicas.
  • Hito Técnico: Se optimizó el software del ESP32 aplicando algoritmos de Debounce (antirrebote) para eliminar los ruidos elásticos de los finales de carrera mecánicos, consiguiendo lecturas de velocidad puras y constantes.

Fase II: Infraestructura de Comunicaciones e IoT (MQTT)

Con el hardware transmitiendo de forma local, el siguiente paso fue enviar los datos hacia el servidor. Para ello se desplegó un broker Mosquitto MQTT. Se realizón un estudio de mapa de calor de la señal WiFi en el laboratorio para ubicar un punto de acceso para asegurar transmisiones estables y con el mínimo de pérdidas.

Fase III: Desarrollo del Backend de Producción (Spring Boot)

Para procesar la telemetría, se construyó una aplicación empresarial con Spring Boot (Java). Se programó un componente oyente (`MqttListener`) encargado de suscribirse al broker de forma asíncrona, capturar las ráfagas de datos entrantes, procesar los strings y mapearlos en objetos de datos organizados.

Fase IV: Sincronización en Tiempo Real y Frontend

  • Se estructuró un sistema de consultas repetitivas HTTP Polling.
  • Se implementó un diseño amigable y a juego con los colores de la Universidad.

Fase V: Módulo de Adquisición de Datos e Históricos (.CSV)

La última gran fase consistió en dotar a la herramienta de utilidad científica. Se integró un panel de control con temporizadores automáticos que permite al operador de la Universidad iniciar la grabación de un ensayo. El backend captura la secuencia exacta de datos, la almacena dinámicamente en memoria y genera un reporte descargable en formato CSV, guardando simultáneamente un registro histórico en la base de datos para consultas posteriores mediante un panel administrativo jerarquizado por roles (ADMIN, KEYUSER, USER).


4. Arquitectura Tecnológica del Sistema

A nivel de software y hardware, el proyecto está compuesto por el siguiente stack:

  • Hardware: Microcontroladores ESP32, Acelerómetro ADXL345, Sensores mecánicos de fin de carrera y Caudalímetros por pulsos.
  • Protocolos de Red: MQTT (Broker Mosquitto) para la captura perimetral
  • Core Backend: Java 17, Spring Boot, Spring Security, Spring Integration MQTT, y JPA / Hibernate.
  • Frontend: HTML5 semántico, Thymeleaf Engine, Vanilla JavaScript

5. Conclusión y Estado Actual

El proyecto se encuentra actualmente en un alto estado de madurez de producción, resolviendo con éxito los problemas iniciales. Representa una solución de bajo coste, alta precisión y estándar industrial (SCADA) lista para ser utilizada en ensayos metalúrgicos controlados dentro de nuestra comunidad universitaria.
